# useConfirmModal

Custom hook to display a confirmation modal via `DPortalContext`.
It handles all internal state (loading, critical code validation) and renders the modal automatically — no JSX needed at the call site.

Requires `DContextProvider` (which includes `DPortalContext`) wrapping your application.

---

## Parameters

### `UseConfirmModalConfig`

| Prop | Type | Required | Default | Description |
| --- | --- | --- | --- | --- |
| `title` | `string` | | `'Are you sure you want to proceed?'` | Modal header title. |
| `message` | `string` | | `'Please confirm if you want to continue with this action.'` | Body text shown inside the modal. |
| `confirmLabel` | `string` | | `'Confirm'` | Label for the confirm button. |
| `cancelLabel` | `string` | | `'Cancel'` | Label for the cancel/close button. |
| `confirmColor` | `ConfirmModalColor` | | `'primary'` | Color variant of the confirm button. |
| `onConfirm` | `() => void \| Promise<void>` | ✅ | — | Async-safe callback executed when the user confirms. The modal stays open (with a loading state) until the promise resolves. |
| `onClose` | `() => void` | | — | Called after the modal closes via the cancel button or the X icon. |
| `critical` | `CriticalConfirmConfig` | | — | Enables a critical confirmation mode that requires the user to type a specific code before confirming. |

### `ConfirmModalColor`

```ts
type ConfirmModalColor = 'primary' | 'danger' | 'warning';
```

### `CriticalConfirmConfig`

| Prop | Type | Required | Default | Description |
| --- | --- | --- | --- | --- |
| `code` | `string` | ✅ | — | The exact string the user must type to enable the confirm button. |
| `codeLabel` | `string` | | `'Confirmation code'` | Label shown above the code input. |
| `inputPlaceholder` | `string` | | — | Placeholder text for the code input. |

---

## Return Values

### `UseConfirmModalReturn`

| Property | Type | Description |
| --- | --- | --- |
| `open` | `() => void` | Opens the confirmation modal. |
| `close` | `() => void` | Programmatically closes the modal and fires `onClose` if provided. |
| `isLoading` | `boolean` | `true` while `onConfirm` is executing. Useful to disable triggers while a confirmation is in progress. |

## Basic example `ExampleOfUse.tsx`

Simple destructive confirmation with a danger color.

<Source
  code={`import { useState } from 'react';
import { DButton, useConfirmModal } from '@dynamic-framework/ui-react';

export function ExampleOfUse() {
  const [result, setResult] = useState('');

  const confirm = useConfirmModal({
    title: 'Delete Item',
    message: 'Are you sure you want to delete this item? This action cannot be undone.',
    confirmLabel: 'Delete',
    cancelLabel: 'Cancel',
    confirmColor: 'danger',
    onConfirm: async () => {
      await deleteItem();
      setResult('Item deleted successfully.');
    },
    onClose: () => {
      setResult('Action cancelled.');
    },
  });

  return (
    <div className="d-flex flex-column align-items-center gap-3">
      <DButton text="Delete Item" color="danger" onClick={confirm.open} />
      {result && <p className="alert alert-info mb-0">{result}</p>}
    </div>
  );
}`}
  language="tsx"
  dark
/>

## Critical confirmation

When a destructive action is irreversible, use `critical` to force the user to type a confirmation code before enabling the confirm button.

<Source
  code={`import { useState } from 'react';
import { DButton, useConfirmModal } from '@dynamic-framework/ui-react';

export function DeleteAccountButton() {
  const [result, setResult] = useState('');

  const confirm = useConfirmModal({
    title: 'Delete Account',
    message: 'This is a permanent action. All your data will be erased and cannot be recovered.',
    confirmLabel: 'Delete Account',
    cancelLabel: 'Cancel',
    confirmColor: 'danger',
    critical: {
      code: 'DELETE ACCOUNT',
      codeLabel: 'Type the confirmation code to proceed',
      inputPlaceholder: 'Type: DELETE ACCOUNT',
    },
    onConfirm: async () => {
      await deleteAccount();
      setResult('Account deleted permanently.');
    },
  });

  return (
    <div className="d-flex flex-column align-items-center gap-3">
      <DButton text="Delete Account" color="danger" onClick={confirm.open} />
      {result && <p className="alert alert-danger mb-0">{result}</p>}
    </div>
  );
}`}
  language="tsx"
  dark
/>

## Using `onClose` callback

`onClose` fires after the modal closes via the cancel button or the X icon. Use it to react to user cancellation.

<Source
  code={`const confirm = useConfirmModal({
  title: 'Archive Record',
  message: 'This record will be moved to the archive. You can restore it later.',
  confirmLabel: 'Archive',
  cancelLabel: 'Keep',
  confirmColor: 'warning',
  onConfirm: async () => {
    await archiveRecord();
  },
  onClose: () => {
    console.log('User dismissed the modal.');
  },
});`}

## Dynamic message per item

Pass a dynamic `message` by calling `useConfirmModal` inside the component that owns the item's data. Each instance is fully independent.

<Source
  code={`function ContactRow({ contact, onDeleted }) {
  const confirm = useConfirmModal({
    title: 'Delete Contact',
    message: \`Are you sure you want to delete \${contact.name} from your contact list?\`,
    confirmLabel: 'Delete',
    cancelLabel: 'Cancel',
    confirmColor: 'danger',
    onConfirm: async () => {
      await deleteContact(contact.id);
      onDeleted(contact.id);
    },
  });

  return (
    <div className="d-flex align-items-center gap-3">
      <span>{contact.name}</span>
      <DButton text="Delete" color="danger" variant="outline" onClick={confirm.open} />
    </div>
  );
}`}
  language="tsx"
  dark
/>
