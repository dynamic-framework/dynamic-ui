# Color Tokens

Dynamic Framework provides semantic color tokens with extended shades (25, 50, 100 … 900) for text, backgrounds, borders, badges, and buttons. Tokens: Primary, Success, Warning, Danger, Info, Secondary, Light, Dark. Light variants are designed for dark backgrounds.

<br />

### Text colors
Use text tokens to communicate state and semantics.

<div className="d-flex flex-wrap gap-3 mb-4">
  <p className="m-0 text-primary">.text-primary</p>
  <p className="m-0 text-success">.text-success</p>
  <p className="m-0 text-warning">.text-warning</p>
  <p className="m-0 text-danger">.text-danger</p>
  <p className="m-0 text-info">.text-info</p>
  <p className="m-0 text-secondary">.text-secondary</p>
  <p className="m-0 text-dark">.text-dark</p>
  <p className="m-0 text-light bg-dark rounded px-2">.text-light (use on dark bg)</p>
</div>

```html
<p class="text-success">Success text</p>
<p class="text-danger">Error text</p>
```

<br />

<div class="bg-info-200">Informational tint</div>

### Badges
Badges support semantic color tokens via text-bg-*.

  <span className="badge text-bg-danger">Danger</span>

<div className="d-flex flex-column gap-2 mb-4">

### Dark mode and utilities
- Prefer .text-light over dark surfaces.
- Combine tokens with dark: for system/forced dark contexts.
See: <a href="/?path=/story/design-system-examples-dark-utilities">Design System/Examples/Dark Utilities</a> and <a href="/?path=/story/design-system-foundations-extended-utilities">Foundations/Extended Utilities</a>.

## Accessibility guidance
- Ensure WCAG AA contrast when pairing text and backgrounds.
- Use semantic tokens consistently for clear communication.
