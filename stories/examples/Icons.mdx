## Icons

The `DIcon` component is a versatile icon component that can be configured to use different icon fonts. It supports both traditional icon fonts and material-style icon fonts.

### Configuration

The `DIcon` component is configured using the `DContextProvider`. The `DContextProvider` accepts an `icon` object with the following properties:

- `familyPrefix`: The prefix for the icon class (e.g., `bi-` for Bootstrap).
- `familyClass`: The class for the icon family (e.g., `bi-icon` for Bootstrap icons).
- `materialStyle`: A boolean flag that determines how the icon is rendered. If `true`, the icon is rendered with the name of the icon in the content of the tag. If `false`, the icon is rendered with the class of the icon font.

<Source
  code={`
<DContextProvider
  icon={{
    familyClass: 'bi-icon',
    familyPrefix: 'bi-',
    materialStyle: false,
  }}
>
  <DIcon name="user" />
</DContextProvider>
  `}
  language="html"
/>


### Material Style Flag

This flag comes in handy when you have icon fonts that need to be included as the content of the tag.

Following example show the output of `DIcon` component with and without the `materialStyle` flag.

<Source
  code={`
  // Disabled Material Style Flag (default)
<i class="bi bi-user"></i>
  `}
  language="html"
/>

<Source
  code={`
  // Enabled Material Style Flag
<i class="bi">user</i>
  `}
  language="html"
/>


## Example Usage


#### Font Awesome Icons

To use Font Awesome icons, configure the `DContextProvider` as follows:

```jsx
<DContextProvider
  icon={{
    familyPrefix: 'fa-',
    familyClass: 'fa-solid',
    materialStyle: false,
  }}
>
  <DIcon name="user" />
</DContextProvider>
```

#### Material Icons

To use Material icons, configure the `DContextProvider` as follows:

```jsx
<DContextProvider
  icon={{
    familyPrefix: '',
    familyClass: 'material-symbols-outlined',
    materialStyle: true,
  }}
>
  <DIcon name="user" />
</DContextProvider>
```
