# Tables

Our framework uses native HTML tables enhanced with [Bootstrap styling](https://getbootstrap.com/docs/5.3/content/tables/) and custom components for advanced features like sorting, pagination, and selection.

## Quick Reference

| Feature | Component/Class | Use Case |
|---------|----------------|----------|
| Basic styling | `.table` | All tables |
| Striped rows | `.table-striped` | Improve readability |
| Hover effect | `.table-hover` | Interactive tables |
| Borders | `.table-bordered` | Emphasize cells |
| Compact | `.table-sm` | Dense data |
| Sorting | `<DTableHead>` | Sortable columns |
| Pagination | `<DPaginator>` | Multi-page data |
| Selection | `useItemSelection()` | Bulk actions |
| Loading | `.placeholder-wave` | Skeleton screens |

## Best Practices

### ✅ Do

- Use `<caption>` for accessibility
- Use semantic HTML (`<thead>`, `<tbody>`, `<tfoot>`)
- Implement pagination for tables with >20 rows
- Show loading states during data fetching
- Make tables responsive on mobile
- Use `DTableHead` for sorting instead of custom solutions

### ❌ Don't

- Nest tables inside tables
- Use tables for layout (use CSS Grid/Flexbox)
- Forget to handle empty states
- Implement sorting without visual indicators
- Create tables without headers
